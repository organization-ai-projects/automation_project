# Contributing to Protocol Macros

Thank you for your interest in contributing to the `protocol_macros` crate! This document provides guidelines for contributing to the project.

## How to Contribute

1. **Fork the Repository**

   - Create a fork of the repository on GitHub.

2. **Clone the Repository**

   - Clone your fork locally:

     ```bash
     git clone https://github.com/your-username/protocol_macros.git
     ```

3. **Create a Branch**

   - Create a new branch for your feature or bug fix:

     ```bash
     git checkout -b feature/my-new-feature
     ```

4. **Make Changes**

   - Implement your changes.
   - Add tests for new features or bug fixes.

5. **Run Tests**

   - Ensure all tests pass:

     ```bash
     cargo test
     ```

6. **Commit Changes**

   - Commit your changes with a descriptive message:

     ```bash
     git commit -m "Add feature: my new feature"
     ```

7. **Push Changes**

   - Push your branch to your fork:

     ```bash
     git push origin feature/my-new-feature
     ```

8. **Create a Pull Request**

   - Open a pull request on GitHub.

## Guidelines

- Follow Rust coding standards.
- Write clear and concise commit messages.
- Ensure all new code is covered by tests.
- Update documentation for any new features.

For more details, refer to the [README](../README.md).
